# XML Schemas (XSD)

This directory holds the ISO 20022 schemas used for full XSD validation of the test
fixtures in CI (`xsd-validate` job).

## Vendored schemas

| File | Message | Status |
|---|---|---|
| `pacs.008.001.08.xsd` | FI to FI Customer Credit Transfer | validated in CI |
| `pacs.002.001.10.xsd` | FI to FI Payment Status Report | validated in CI |
| `head.001.001.02.xsd` | Business Application Header (BAH) | vendored for upcoming BAH support |

## Provenance and licensing

These are the **base ISO 20022 schemas**, generated by the ISO 20022 Standards
Editor. Canonical source: [iso20022.org](https://www.iso20022.org/iso-20022-message-definitions)
(2019-era versions, in the messages archive). The copies here were obtained from
[moov-io/fednow20022](https://github.com/moov-io/fednow20022) (Apache-2.0), which
redistributes them publicly. The ISO 20022 IPR policy allows free use and
implementation of the message definitions.

**FedNow-specific schema variants and usage guidelines** are distributed by the
Federal Reserve through the MyStandards platform (free account, registration
required). **Do not commit files from MyStandards here** unless their license
explicitly allows redistribution — check before vendoring anything from that portal.
The base ISO schemas in this directory are a superset: a message that is valid
against the FedNow usage guideline is also valid against the base schema, so CI
validation against these files is a correct (if not complete) check.

## Adding a schema

Drop the XSD here with its canonical filename and add a line to the `xsd-validate`
job in `.github/workflows/ci.yml` mapping the fixture glob to the schema.
